Site Documentation

Implementation notes for the Blog page, navigation and dropdown styling, SEO elements, repository and deployment practices.

Blog page and navigation

  • New page: blog.html with full on-page SEO (title, description, canonical, Open Graph, Twitter Card, JSON-LD Blog + Breadcrumbs).
  • Desktop nav: Added Blog link on all pages.
  • Mobile dropdown: blue/white background with bold appearance. If using Fonseca, load via @font-face or a font service; otherwise it falls back to Inter.
  • Sitemap: sitemap.xml includes blog.html and privacy-policy.html.

SEO checklist (new pages)

  • Unique <title> and <meta name="description">.
  • Canonical: <link rel="canonical" href="...">.
  • Open Graph + Twitter Card tags with a representative image.
  • JSON-LD schema matching the content (e.g., Article for posts, Blog for listing).
  • Link the page in nav/footer; add to sitemap.xml.

Where to edit

  • Desktop nav: search for ml-10 flex items-baseline in each HTML.
  • Mobile menu: search for id="mobile-menu" to edit items and styles.
  • Footer links: search for Quick Links on each page.
  • Blog cards: edit the cards in blog.html.

Repository & deployment

  • GitHub: github.com/mikdigitalofficial/portfolio
  • Issues: /issues
  • PRs: /pulls
  • After adding pages, update sitemap.xml, commit, and push to main.
  • Ensure canonical URLs point to the production domain.

Cookies policy

I use cookies to enhance UX, measure analytics, and support marketing. Strictly necessary cookies are always on; others are opt-in.

  • Consent: The banner stores preferences in localStorage under mik_cookie_consent_v1.
  • Categories: Analytics and Marketing can be toggled in the banner via “Cookie settings.”
  • Change preferences: Clear site data or run localStorage.removeItem('mik_cookie_consent_v1') to reshow the banner.
  • Policy: Full details in Privacy Policy.
